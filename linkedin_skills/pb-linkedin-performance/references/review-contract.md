# Review Contract

Use this file when producing a performance review from LinkedIn analytics exports.

## Primary Goal

Turn exported LinkedIn analytics into useful editorial feedback without collapsing into vanity-metric thinking.

## What Good Reviews Do

- separate what is known from what is missing
- describe the recent window in plain terms
- compare posts only when the comparison is fair
- recommend a few constrained experiments
- preserve the PersonalBrand voice and positioning

## What Bad Reviews Do

- overfit to one post
- confuse impressions with content quality
- recommend gimmicks or hype patterns
- push the user toward generic founder-thought-leader language
- use demographics tables as if they directly dictate the content strategy

## Recommendation Guardrails

Allowed recommendation types:

- clarify the first 1 to 2 lines
- sharpen the external reader payoff
- rebalance lane mix
- test more concrete scenes
- preserve what appears authentic while improving packaging

Forbidden recommendation types:

- engagement bait
- fake controversy
- forced CTAs
- fake authority
- rewriting the whole voice around performance
- claiming causality from a single export window

## Preferred Metrics Priority

Use metrics in this order:

1. impressions
2. aggregate engagements
3. engagement rate when computable
4. follower delta
5. demographics as supporting context only

If a metric is absent from the export, say that clearly.
